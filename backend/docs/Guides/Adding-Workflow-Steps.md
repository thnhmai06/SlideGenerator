# How-to: Adding Workflow Steps

This guide explains how to create, implement, and register a new step in the slide generation pipeline.

## 1. Create the Step Body

All steps live in `SlideGenerator.Generator/Application/Steps/`.

Inherit from `StepBody` (synchronous) or `StepBodyAsync` (asynchronous). Use **Primary Constructors** for dependency injection.

```csharp
public sealed class MyNewStep(IMyService myService) : StepBodyAsync
{
    // Inputs from the workflow data
    public string SomeInput { get; init; } = null!;

    public override async Task<ExecutionResult> RunAsync(IStepExecutionContext context)
    {
        // 1. Access the global workflow data
        var data = (GeneratingContext)context.Workflow.Data;

        // 2. Perform logic
        await myService.DoSomethingAsync(SomeInput);

        // 3. Signal completion
        return ExecutionResult.Next();
    }
}
```

## 2. Register the Step

Open `SlideGenerator.Generator/Injection/Registration.cs` and add your step inside `AddGeneratorServices()`:

```csharp
services.AddTransient<MyNewStep>();
```

## 3. Map the Step in the Workflow

Open `SlideGenerator.Generator/Application/Workflows/GeneratingWorkflow.cs`. Find the appropriate Phase (A, B, or C) and insert your step using the Fluent API.

```csharp
.Then<MyNewStep>()
    .Input(step => step.SomeInput, data => data.SpecificField)
```

### Mapping for Collections
If your step runs inside a `.ForEach()` block, map the input from the `context.Item`:

```csharp
.ForEach(data => data.MyCollection)
    .Do(x => x
        .StartWith<MyNewStep>()
        .Input(step => step.SomeInput, (data, context) => (string)context.Item)
    )
```

## Best Practices

1. **Idempotency**: Ensure the step can be safely re-run if the workflow resumes after a crash (e.g., check if a file already exists before creating it).
2. **Error Handling**: Wrap logic in `try-catch` and log errors to `data.Logger`. Do not let exceptions bubble up to the engine unless the entire workflow must stop.
3. **Throttling**: If your step is resource-intensive, inject `IGateLocker` and acquire a gate.
4. **Logging**: Use `data.Logger.BeginScope` to attach context (like row index) to all log entries generated by the step.
